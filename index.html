Going back to the classic editor

The staff at WordPress.com have been working towards a unified dashboard to manage all your blogs in one place on any device. However, many users feel that it falls short of the classic dashboards, me included.

I'm sure most people would be fine with this as long as we are given a choice of interface, and that that choice remains consistent through our experience. However, on March 13, 2015, the preference setting for the choice of editor, which was implemented by a cookie a link back to the classic editor, was removed. Cue the uproar of the mass of angry and frustrated users when their cookies expire. Pages of complaints have been posted in the forums in response to the editor essentially being forced on us.

If you like the new editor, then by all means use it because it's all about choice. But if you're also displeased with the new editor and would like to continue using the classic editor, you'll be glad to know that there are a few workarounds. They are listed here in order of overall ease of installation and use.

Automatic redirect: using a user script

I argue that this is the easiest because the only hard part is the initial installation. After that, everything redirects automatically without your input and you can almost forget you're using a script.

This method works best for Mozilla Firefox, Google Chrome, Opera, and their derivatives. If you are using Safari or Internet Explorer, it may be easier to use the bookmarklet method instead. If you don't know what browser you're using, you can go to whatismybrowser.com to find out.

Follow these steps to install the script:

  1. Install an add-on for your browser to run user scripts:

    Firefox and derivatives
    Greasemonkey or Scriptish
    Chrome and derivatives
    Tampermonkey
    Opera
    Tampermonkey, Violentmonkey
    Safari
    NinjaKit
    Internet Explorer
    TrixIE

  2. Install the script by clicking the Install link below:

    [Install]

Any time you visit a link to the new editor, you will be redirected to the classic one. Note that because of the way the script works, you may see the new editor briefly, and it will load completely before redirecting for private and Jetpack-enabled blogs.

Manual redirect: using a bookmarklet

This method is the easiest to set up and very simple to use. Drag the link below into your bookmarks:

Edit in classic editor

Or, if you can't drag, create a new bookmark with the following URL:

javascript:window.location.replace(document.getElementsByClassName('switch-to-classic')[0].children[0].href)

When you're on the new editor, click on the bookmark you made to edit your post in the classic editor. Note that any unsaved changes you made to the post will be lost after clicking the bookmark.

Explicit request: using bookmarks for each site

This method is painstakingly tedious and time-consuming; I recommend using one of the methods above if you can. This is included here for completeness, especially because it's the only officially endorsed solution.

External editing: using WordPress.org

This method is a lot more trouble than it's worth. The idea is to set up a WordPress.org installation somewhere, write (and possibly publish) your posts there, and manually copy your post's content to WordPress.com.

If you're doing this, you may as well just move to your WordPress.org installation and not use WordPress.com.
